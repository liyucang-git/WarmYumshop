# 暖圆小铺 - 微信小程序需求文档

## 项目概述

### 基本信息
- **项目名称**：暖圆小铺
- **项目类型**：微信小程序
- **项目定位**：一款聚焦家庭场景的轻量工具小程序，家庭餐桌规划，用简洁的设计陪伴每个家庭的日常
- **开发周期**：待定
- **开发团队**：待定

### 产品愿景
通过温暖简约的数字体验，帮助家庭记录和分享私房菜谱，留住美食时光，让每一个家庭都能保留属于自己的味道记忆。

---

## 一、产品定位与目标用户

### 1.1 产品定位
- **核心价值**：简单易用的家庭私房菜谱记录工具
- **设计理念**：简洁、温暖、实用
- **目标场景**：家庭日常饮食记录与分享

### 1.2 目标用户画像
- **主要用户**：25-45岁的家庭主妇/煮夫
- **次要用户**：需要管理家庭饮食的年轻夫妻
- **用户特征**：
  - 注重家庭生活质量
  - 希望有规律的饮食计划
  - 对新工具有一定的接受度
  - 追求生活便利性和品质感

## 二、核心功能需求

### 2.1 个人中心模块

#### 2.1.1 用户登录
**功能描述**：用户首次使用需要通过微信授权登录

**详细需求**：
- 使用微信wx.login获取用户openid
- 通过openid查询云数据库判断用户是否已存在
- 如用户已存在：从数据库获取用户信息，登录成功
- 如用户不存在：跳转到"完善个人信息"页面，引导用户手动填写或选择头像和昵称
  - 使用 `<button open-type="chooseAvatar">` 让用户选择头像
  - 使用 `<input type="nickname">` 让用户输入昵称
  - 填写完成后存入数据库，记录首次登录小程序时间

**验收标准**：
- 微信登录流程正常
- 用户可手动选择头像和输入昵称
- 登录状态正确保存
- 未完善信息的用户引导流程完整

#### 2.1.2 个人中心页面
**功能描述**：用户查看和编辑个人信息的入口

**详细需求**：
- 显示用户头像和昵称
- **有家庭时**：
  - 显示当前所属家庭名称
  - 显示用户角色（家庭创建者/成员）
  - 点击家庭区域可进入家庭管理页
- **无家庭时**：
  - 显示"创建或加入家庭"引导区域
  - 提供创建家庭按钮
  - 提供加入家庭按钮
- 提供退出登录入口

**验收标准**：
- 用户信息正确展示
- 家庭状态判断逻辑正确
- 页面加载流畅
- 退出登录功能正常

### 2.2 家庭管理模块

#### 2.2.1 创建家庭
**功能描述**：用户可以创建自己的家庭

**详细需求**：
- 输入家庭名称（必填，2-20个字符）
- 创建者自动成为家庭管理员
- 创建成功后自动加入该家庭
- 家庭创建者可解散家庭

**验收标准**：
- 家庭名称验证通过
- 创建成功后正确跳转
- 家庭信息正确保存到云端

#### 2.2.2 加入家庭
**功能描述**：用户可以输入家庭邀请码加入已有家庭

**详细需求**：
- 输入家庭邀请码
- 验证家庭是否存在
- 验证通过后加入家庭
- 同一用户只能加入一个家庭

**验收标准**：
- 邀请码格式验证
- 家庭不存在时提示错误
- 成功加入后更新用户状态

#### 2.2.3 家庭邀请码
**功能描述**：家庭成员可查看和分享家庭邀请码

**详细需求**：
- 邀请码在创建家庭时自动生成，存储在家庭表中
- 显示当前家庭的邀请码
- 支持复制邀请码
- 邀请码与家庭ID绑定
- 邀请码长期有效

**验收标准**：
- 邀请码正确显示
- 复制功能正常
- 邀请码可正常使用

#### 2.2.4 家庭成员管理
**功能描述**：家庭管理员可以管理成员，其他成员仅可查看

**详细需求**：
- 查看家庭成员列表
- 显示成员头像、昵称、加入时间（加入时间存储在数据库中）
- 管理员可移除成员（非本人），也可解散家庭
- 显示成员数量

**验收标准**：
- 成员列表正确显示
- 移除成员功能正常
- 权限控制正确

#### 2.2.5 离开家庭
**功能描述**：家庭成员可以主动离开家庭

**详细需求**：
- 确认离开提示
- 离开后清空用户家庭关联
- 家庭创建者不可离开，需先解散家庭

**验收标准**：
- 离开确认弹窗正常
- 离开后状态正确更新
- 权限控制正确

### 2.3 菜品管理模块

#### 2.3.1 添加菜品
**功能描述**：用户可以创建新的菜品记录

**详细需求**：
- 菜品名称输入（必填，1-50个字符）
- 菜品分类选择（必填，支持多选分类）
- 菜品描述输入（可选，0-200个字符）
- 菜品图片上传（必填，支持拍照和相册选择）
- 添加成功后提示用户

**验收标准**：
- 所有必填项未填写时，提交按钮不可用
- 图片上传后可预览和重新选择
- 添加成功后菜品立即显示在列表中

#### 2.3.2 菜品列表展示
**功能描述**：以简洁清晰的列表/网格形式展示所有已添加的菜品

**详细需求**：
- 支持列表视图和网格视图切换
- 每个菜品显示：图片、菜名、分类标签
- 支持按分类筛选菜品
- 支持按添加时间排序（最新/最旧）
- 支持搜索菜品（按菜名）

**验收标准**：
- 列表加载流畅，无明显卡顿
- 图片加载失败时显示占位图
- 筛选和搜索结果准确

#### 2.3.3 菜品详情查看
**功能描述**：查看菜品的详细信息

**详细需求**：
- 大图展示菜品图片（可点击放大）
- 显示菜名、分类、描述
- 显示添加/更新时间
- 提供编辑和删除入口

**验收标准**：
- 图片清晰，支持手势放大缩小
- 信息展示完整且层次分明
- 编辑和删除操作有确认提示

#### 2.3.4 菜品编辑与删除
**功能描述**：对已有菜品进行修改和删除操作

**详细需求**：
- 编辑：可修改菜品的所有信息
- 删除：删除前弹出确认提示
- 删除成功后从列表中移除
- **云存储垃圾回收**：删除菜品时，同步调用云函数删除云存储（Cloud Storage）中对应的 fileID 图片文件，避免占用额外空间
- **权限控制**：菜品属于家庭，个人只能看到和管理自己家庭的菜品；非家庭成员无法看到菜品

**验收标准**：
- 编辑后信息及时更新
- 删除操作可撤销（可选）
- 删除操作有二次确认
- 非家庭成员无法看到编辑/删除按钮
- 删除菜品时原图片文件同步从云存储中清除

### 2.4 图片上传模块

#### 2.4.1 拍照上传
**功能描述**：直接调用微信相机拍照并上传菜品图片

**详细需求**：
- 调用微信相机API
- 支持前后摄像头切换
- 支持拍照后预览
- 自动压缩图片（限制大小）

**验收标准**：
- 相机调用正常
- 拍照后图片显示清晰
- 图片大小符合限制（建议<5MB）

#### 2.4.2 相册选择
**功能描述**：从微信相册选择已有图片上传

**详细需求**：
- 支持选择单张图片

**验收标准**：
- 相册选择正常
- 图片加载速度合理
- 图片大小符合限制

#### 2.4.3 图片预览与管理
**功能描述**：上传后可在菜品详情中预览图片，支持更换和删除

**详细需求**：
- 支持点击放大预览
- 支持手势缩放查看
- 可重新上传替换图片（替换前自动删除原图片）
- 可删除图片（需重新上传）

**验收标准**：
- 预览流畅无卡顿
- 手势缩放自然
- 更换和删除操作有确认提示
- 替换图片时原文件从云存储中同步清除

### 2.5 菜品分类模块

#### 2.5.1 内置分类
**功能描述**：提供常用菜品分类供用户选择

**内置分类列表**：
- 家常菜

- 甜品

**功能描述**：可按分类查看对应菜品

**详细需求**：
- 首页显示分类标签栏
- 点击分类标签筛选对应菜品
- 支持"全部"选项显示所有菜品

**验收标准**：
- 筛选结果准确
- 标签切换流畅

### 2.6 数据持久化模块

#### 2.6.1 云端存储
**功能描述**：所有数据存储在微信云开发云端，保障数据安全和多设备同步

**详细需求**：
- 使用微信云数据库存储所有数据
- 菜品数据包括：ID、菜名、分类（支持多个分类）、描述、图片（云存储ID）、创建时间、更新时间、所属家庭ID
- 默认分类配置在前端代码常量文件中（如：家常菜、汤羹、主食、甜品、凉菜、热菜、面食、其他），避免频繁查询数据库
- 用户数据存储（用户ID、openid、昵称、头像、所属家庭ID）
- 家庭数据存储（家庭ID、家庭名称、创建者ID、邀请码、成员列表）

**数据结构设计**：
- **用户表（users）**：_id, openid, nickname, avatarUrl, familyId, role(creator/member), joinTime, createTime
- **家庭表（families）**：_id, name, creatorId, inviteCode, members(包含用户ID和加入时间), createTime
- **菜品表（dishes）**：_id, name, categories(支持多个分类), description, imageUrl, familyId, createTime, updateTime
- **分类配置（constants/categories.js）**：前端常量文件，固定数组["家常菜", "汤羹", "主食", "甜品", "凉菜", "热菜", "面食", "其他"]

**验收标准**：
- 数据读写速度快（云数据库查询性能优良）
- 数据不丢失（云端自动备份）
- 数据结构合理，便于查询
- 家庭间数据完全隔离

#### 2.6.2 数据安全
**功能描述**：保障用户数据安全和隐私

**详细需求**：
- 用户数据与openid绑定
- 家庭数据仅家庭成员可访问
- 菜品数据按家庭隔离存储
- 云存储图片设置合理的访问权限

**验收标准**：
- 非家庭成员无法访问家庭数据
- 数据隔离正确
- 隐私保护有效

### 2.7 用户界面设计

#### 2.7.1 设计风格
- **整体风格**：简洁、温暖、现代
- **主色调**：橙色系（温馨感）
- **辅助色**：白色、浅灰色
- **字体**：PingFang SC

#### 2.7.2 页面布局

**一级页面（底部导航）**：
- **首页**：顶部导航栏 + 分类筛选区 + 菜品列表区 + 悬浮添加按钮 + 底部导航栏
- **个人中心页**：用户信息区 + 家庭信息区 + 家庭操作区 + 其他操作区

**二级页面（通过一级页面进入）**：
- **详情页**：顶部导航栏 + 菜品图片区 + 菜品信息区 + 操作按钮区
- **添加页**：顶部导航栏 + 表单区域 + 操作按钮
- **家庭管理页**：家庭信息区 + 成员列表区 + 操作按钮区
- **创建/加入家庭页**：Tab切换 + 表单区域 + 操作按钮

#### 2.7.3 交互设计
- 流畅的页面切换动画
- 友好的操作反馈（Toast提示）
- 合理的按钮点击态
- 优雅的加载状态

**全局提示规范（Toast）**：
- **成功提示**：显示绿色图标 + 文案，停留时间 2 秒（如："保存成功"）
- **失败提示**：显示红色图标 + 文案，停留时间 3 秒（如："操作失败，请重试"）
- **警告提示**：显示橙色图标 + 文案，停留时间 2.5 秒（如："请填写完整信息"）
- **加载提示**：显示Loading图标 + 文案，无自动关闭（如："保存中..."）
- **Toast弹窗位置**：页面垂直居中
- **按钮点击态**：透明度变化（0.6），持续时间 100ms

- **网络异常与弱网处理**：
  - 页面加载时显示骨架屏或Loading动画
  - 上传图片等耗时操作必须显示Loading遮罩，防止用户重复提交
  - 网络请求失败时显示友好提示（如："网络不稳定，请稍后重试"）
  - 无网络时显示离线提示页面

#### 2.7.4 空状态设计（Empty State）
**详细需求**：
- 首页菜品列表为空时：显示温馨插画 + 引导文案（如："快来添加第一道拿手好菜吧"）+ 引导添加按钮
- 家庭成员列表为空时：显示引导文案 + 邀请家庭成员按钮
- 搜索无结果时：显示"暂无相关内容"提示
- **空状态设计原则**：插画风格与小程序整体温暖、简约风格一致；文案温馨亲切，引导用户下一步操作

**验收标准**：
- 空状态页面美观、温馨
- 引导按钮可正常跳转
- 文案清晰传达下一步操作

## 三、技术架构设计

### 3.1 技术栈选择

#### 3.1.1 前端技术
- **小程序框架**：微信原生小程序开发框架
- **开发语言**：JavaScript（或TypeScript）
- **页面结构**：WXML
- **页面样式**：WXSS
- **组件库**：TDesign 小程序组件库

#### 3.1.2 后端技术
- **云开发服务**：微信云开发（腾讯云）- 核心存储方案
- 云数据库：存储用户、家庭、菜品、分类数据
- 云存储：存储菜品图片
- 云函数：处理业务逻辑（用户登录、家庭管理等）
- **不再使用本地存储方案**，所有数据通过云端存储

#### 3.1.3 开发工具
- **IDE**：微信开发者工具
- **版本控制**：Git
- **设计工具**：Figma/Sketch（可选）

### 3.2 系统架构

#### 3.2.1 整体架构
采用经典的微信小程序三层架构：
- **视图层**：WXML + WXSS 负责页面展示和用户交互
- **逻辑层**：JavaScript/TypeScript 处理业务逻辑和数据管理
- **数据层**：微信云数据库负责数据持久化

#### 3.2.2 模块划分
- **UI组件模块**：基于 TDesign 构建的可复用组件
- **服务层模块**：用户服务、家庭服务、菜品服务、图片服务、分类服务
- **工具层模块**：数据验证、格式化、云数据库封装等工具函数
- **配置模块**：路由配置、API配置、主题配置

### 3.3 数据流设计
用户操作 → 事件处理 → 业务逻辑处理 → 云数据库存储 → 界面更新

## 五、性能与安全性要求

### 5.1 性能要求
- 首页加载时间 < 2秒
- 图片加载速度合理（建议使用懒加载）
- 页面切换流畅（60fps）
- 数据存储响应时间 < 100ms

### 5.2 安全性要求
- 用户数据加密存储（可选）
- 图片上传大小限制
- 输入内容过滤，防止XSS攻击
- 合理的权限控制

### 5.3 兼容性要求
- 支持微信最新版本
- 适配iOS和Android平台
- 适配不同屏幕尺寸
